# Support

## Questions and research-methodology discussions

Use GitHub Discussions after it is enabled. Questions about source tiers, causal bridges, sector playbooks, and provider design are better suited to Discussions than bug reports.

## Bugs

Open a bug report using the issue form. Include the ticker, research profile, budget mode, operating system, Python version, visible error, and a redacted provider-health summary.

Never include API keys, proxy credentials, licensed full-text reports, local databases, or confidential investment research.

## Security

Follow [SECURITY.md](SECURITY.md) and use private vulnerability reporting for credential, source-integrity, or local API exposure concerns.

## Investment use

The project does not provide investment advice or trade execution. Maintainers cannot validate a user's investment decision or guarantee provider coverage.
