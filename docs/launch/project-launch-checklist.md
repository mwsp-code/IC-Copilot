# Reusable Project Launch Checklist

## Product

- One memorable problem and one primary workflow.
- Hosted no-signup demo.
- Honest failure and empty states.
- Public benchmark or reproducible proof.
- Clear privacy, security, and licensing policy.

## Repository

- Conversion-focused README with screenshot.
- Three-command quickstart.
- License, contributing guide, Code of Conduct, security policy, roadmap, and changelog.
- Issue forms and pull-request template.
- CI across supported runtimes.
- Tagged release with screenshots and known limitations.

## Community

- Five scoped `good first issue` tasks.
- One pinned welcome discussion.
- One roadmap poll.
- Response target for issues and pull requests.
- Explicit extension points and fixture expectations.

## Distribution

- Analyst/user story.
- Engineering architecture story.
- Demo walkthrough.
- Benchmark post.
- Four-week release and content cadence.

## Measurement

Record weekly:

- GitHub unique visitors and referrers.
- Live demo sessions and completed demo loads.
- Stars per 100 repository visitors.
- Clones and forks.
- Issues opened by external users.
- First-time contributors and merged pull requests.
- Returning demo users.
